webpack
сборщик проекта, где мы указываем входную точку для приложения (один файл) и притягивает в файл все модули, объединяя их в один файл
Babel это хрень, которая современный код js переводит в старые версии для лучшей поддержки браузерами
Scss препроцессор
Eslint хрень, которая следит за чистотой кода
    webpack должен поддерживать 2 режима сборки проекта:
        -Dev mode
            В режиме dev mode идет поддержка:
                Dev Server (автоматически перезагружает окно браузера), 
                SourceMaps (исходные карты, можно увидеть в каком блоке какой код), 
                Eslint (следит за чистотой кода, поддержание в едином стиле), 
                Not Minified JS & CSS (не минифицируем код, чтобы всё было видно), 
        -Prod mode
            в прод моде всё наоборот. там ничего не поддерживается и всё минифицированно.
                Minified Code (максимально сжимает код)
    GIT (git flow способ взаимодействия в системе контроля версий)
        Production (готовая, выпущенная версия, которую видит пользователь) 
            1.0.0
        Development (версия разработки. Её пользователь не видит)
            1.0.2 (1- координально большие изменения. очень сильно отличается от предыдущей
                   0- более значительные изменения, например добавления блоков, тул бара и т.д.
                   2- версия с незначительными изменениями, например изменить цвет кнопки)
            есть версия в разработке. нужно внедрить новую фичу. делается бранч той версии в разработке (копия) и в ней уже делается фича.
            Потом версия с фичей пытается влиться в версию которая в разработке (пулреквест PR)
